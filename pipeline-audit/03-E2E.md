# 03 — End-to-End Test, Every Combination

## What "run it" could and couldn't mean here — read this first

The brief asks to **run** every combination on dev/staging and record PASS/PARTIAL/
FAIL. I drew a hard line at what is **safe and feasible** in this environment:

**Executed for real (deterministic / local):**
- ✅ **Stitch graph builder** — ran `vitest` over the render suites. **383 tests
  pass**, including F01–F12 compiling `buildSeamlessCommand` and asserting real
  `xfade`/`acrossfade` joins, clamped durations, offsets, audio mix, title overlays.
  → The stitch *logic* is proven correct offline.
- ✅ **Pipeline type/contract suites** — `pipeline-types` (43), `pipeline-
  comprehensive` (176), `audio-pipeline` (17) pass.
- ✅ **Python engine runnability** — ran `python3 -m breakout_pipeline.cli` →
  `ModuleNotFoundError: No module named 'numpy'`. Proven it cannot render here.

**NOT executed (would be unsafe / infeasible):** the full cloud chain (mode-router →
hollywood-pipeline → Replicate/OpenAI → seamless-stitcher) cannot be run end-to-end
without: (a) a live Supabase project + deployed edge functions, (b) paid API keys
(OpenAI, Replicate veo/kling/sora/seedance/wan, ElevenLabs), and (c) creating real
`movie_projects` rows and **spending real credits / money on every clip**. Running it
per-combination is an **outward-facing, billable** action with no staging sandbox
wired up in this repo. Per the brief's "dev/staging only" and the standing rule to
confirm costly outward actions, I did **not** fire live generations.

So the per-combination verdicts below are **code-traced** (every handoff verified in
`02-STAGES.md`), labeled `TRACED`, with the deterministic executions above as
supporting evidence. Where a stage is proven broken by code, the verdict is FAIL/
PARTIAL with the exact `file:function` break point — those don't need a live run to
be certain.

> To actually fire live e2e, see `04-GAPS.md §"How to run a real e2e"` for the
> minimal harness (one `mode-router` invoke against staging + poll
> `movie_projects.status`). It's ~30 lines but needs staging creds + budget sign-off.

## Cross-cutting defects that touch *every* PASS row

These don't fail a combination outright but degrade or time-bomb all of them:

1. **Final-film URL expiry (G2)** — every stitched output is a 24h signed URL stored
   as `movie_projects.video_url`; reopening a completed project after a day → 404.
   Hits combinations 1–13.
2. **No edit→re-render propagation (G4)** — editing a script/shot never invalidates
   downstream clips/frames. Hits any post-generation edit on 1–13.
3. **Continuity scoring not load-bearing (G5)** — seams rely on a flat `score>=75`
   gate, no SSIM/pHash seam metric; visible-seam risk across all multi-clip outputs.
4. **External-service dependency (G6)** — correctness of clip gen + stitch rides on
   live Replicate model versions (`kling-v3`, `seedance-2.0`, `veo-3-fast`,
   `sora-2`, `wan-2.5`, ffmpeg cog `efd0b79b…`) + valid keys; unverifiable offline.
